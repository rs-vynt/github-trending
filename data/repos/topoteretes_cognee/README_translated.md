## Giới thiệu về Cognee

Cognee là một nền tảng bộ nhớ AI mã nguồn mở dành cho các Agent AI. Tiêu thụ dữ liệu ở bất kỳ định dạng nào và Cognee liên tục xây dựng một đồ thị tri thức tự lưu trữ mang lại cho các agent của bạn bộ nhớ dài hạn, duy trì liên tục qua các phiên làm việc. Cognee kết hợp nhúng vector (vector embeddings), suy luận đồ thị và việc tạo bản thể học dựa trên khoa học nhận thức để làm cho các tài liệu có thể tìm kiếm được bằng ý nghĩa và được kết nối bởi các mối quan hệ sẽ phát triển cùng với kiến thức của bạn.

:star: _Hãy giúp chúng tôi tiếp cận nhiều nhà phát triển hơn và phát triển cộng đồng cognee. Hãy thả sao cho repo này!_

:books: _Kiểm tra [tài liệu](https://docs.cognee.ai/getting-started/installation#environment-configuration) chi tiết của chúng tôi để biết về việc thiết lập và cấu hình._

:crab: _Có sẵn dưới dạng plugin cho OpenClaw của bạn — [cognee-openclaw](https://www.npmjs.com/package/@cognee/cognee-openclaw)_

✴️ _Có sẵn dưới dạng plugin cho Claude Code của bạn — [claude-code-plugin](https://github.com/topoteretes/cognee-integrations/tree/main/integrations/claude-code)_



### Tại sao nên sử dụng Cognee:

- Dễ dàng xây dựng Bộ não Công ty - hợp nhất dữ liệu từ nhiều nguồn khác nhau ở một nơi và hỗ trợ các Agent bằng kiến thức miền (domain knowledge) của bạn
- Cơ sở hạ tầng kiến thức — tiêu thụ thống nhất, tìm kiếm đồ thị/vector, chạy cục bộ, cơ sở bản thể học, đa phương thức
- Agent liên tục và học hỏi - học từ phản hồi, quản lý ngữ cảnh, chia sẻ kiến thức giữa các agent
- Agent Đáng tin cậy và Đảm bảo - cách ly agent/người dùng đa khách hàng, khả năng truy xuất nguồn gốc, trình thu thập OTEL, dấu vết kiểm toán

## Bắt đầu Nhanh

Hãy thử Cognee chỉ với vài dòng mã.

### Điều kiện Tiên quyết

- Python 3.10 đến 3.14

### Bước 1: Cài đặt Cognee

Bạn có thể cài đặt Cognee bằng **pip**, **poetry**, **uv**, hoặc trình quản lý gói Python ưa thích của bạn.

```bash
uv pip install cognee
```

### Bước 2: Cấu hình LLM
```python
import os
os.environ["LLM_API_KEY"] = "YOUR OPENAI_API_KEY"
```
Thay vào đó, hãy tạo một tệp `.env` sử dụng [mẫu](https://github.com/topoteretes/cognee/blob/main/.env.template) của chúng tôi.

Để tích hợp các nhà cung cấp LLM khác, hãy xem [Tài liệu về Nhà cung cấp LLM](https://docs.cognee.ai/setup-configuration/llm-providers) của chúng tôi.

### Bước 3: Chạy Pipeline

API của Cognee cung cấp cho bạn bốn thao tác — `remember` (ghi nhớ), `recall` (gợi nhớ), `forget` (quên), và `improve` (cải thiện):

```python
import cognee
import asyncio


async def main():
    # Lưu trữ vĩnh viễn vào đồ thị tri thức (chạy add + cognify + improve)
    await cognee.remember("Cognee biến tài liệu thành bộ nhớ AI.")

    # Lưu trữ trong bộ nhớ phiên (cache nhanh, đồng bộ với đồ thị ở chế độ nền)
    await cognee.remember("Người dùng thích giải thích chi tiết.", session_id="chat_1")

    # Truy vấn với định tuyến tự động (tự động chọn chiến lược tìm kiếm tốt nhất)
    results = await cognee.recall("Cognee làm gì?")
    for result in results:
        print(result)

    # Truy vấn bộ nhớ phiên trước, dự phòng sang đồ thị nếu cần
    results = await cognee.recall("Người dùng thích gì?", session_id="chat_1")
    for result in results:
        print(result)

    # Xóa khi xong
    await cognee.forget(dataset="main_dataset")


if __name__ == '__main__':
    asyncio.run(main())

```

### Sử dụng Cognee CLI

```bash
cognee-cli remember "Cognee biến tài liệu thành bộ nhớ AI."

cognee-cli recall "Cognee làm gì?"

cognee-cli forget --all
```

Để mở giao diện người dùng (UI) cục bộ, chạy:
```bash
cognee-cli -ui
```

> **Lưu ý:** Máy chủ MCP được khởi chạy bằng `cognee-cli -ui` sẽ chạy bên trong một container Docker.
> Cần có Docker Desktop, Colima hoặc bất kỳ runtime tương thích OCI nào có CLI `docker` đang hoạt động. Xem [Thiết lập Docker & Colima](docs/docker-colima-setup.md) để biết chi tiết.

## Chạy với Docker

Thích dùng container? Cognee phát hành các image dựng sẵn lên Docker Hub trong mỗi lần đẩy code lên nhánh `main`:
[`cognee/cognee`](https://hub.docker.com/r/cognee/cognee) (máy chủ API) và
[`cognee/cognee-mcp`](https://hub.docker.com/r/cognee/cognee-mcp) (máy chủ MCP).

### Tùy chọn A — Docker Compose (xây dựng từ mã nguồn)

Clone repo, tạo một tệp `.env` với ít nhất là `LLM_API_KEY`, sau đó:

```bash
cp .env.template .env   # sau đó chỉnh sửa .env và thiết lập LLM_API_KEY

# Khởi chạy máy chủ API (http://localhost:8000)
docker compose up

# Các profile tùy chọn (kết hợp tùy ý):
docker compose --profile ui up        # + frontend trên http://localhost:3000
docker compose --profile mcp up       # + máy chủ MCP trên http://localhost:8001
docker compose --profile postgres up  # + Postgres/PGVector
docker compose --profile neo4j up     # + Neo4j
```

> Các dịch vụ `cognee` và `cognee-mcp` xuất ra các cổng host khác nhau (`8000` so với `8001`),
> nên bạn có thể chạy cả hai cùng lúc.

### Tùy chọn B — Tải image dựng sẵn (không cần clone)

```bash
# Tạo một tệp .env tối giản ở thư mục hiện tại
echo 'LLM_API_KEY="YOUR_OPENAI_API_KEY"' > .env

# Máy chủ API
docker run --env-file ./.env -p 8000:8000 --rm -it cognee/cognee:main

# Máy chủ MCP (Giao thức HTTP)
docker pull cognee/cognee-mcp:main
docker run -e TRANSPORT_MODE=http --env-file ./.env -p 8000:8000 --rm -it cognee/cognee-mcp:main
```

Xem [README của máy chủ MCP](cognee-mcp/README.md) về các giao thức truyền tải SSE/stdio, các phần bổ sung tùy chọn và cấu hình client MCP.

## Sử dụng với các Agent AI

### Claude Code

Cài đặt [Plugin bộ nhớ Cognee](https://github.com/topoteretes/cognee-integrations/tree/main/integrations/claude-code) để cung cấp cho Claude Code bộ nhớ duy trì liên tục qua các phiên làm việc. Plugin tự động ghi nhận các lệnh gọi công cụ (tool calls) vào bộ nhớ phiên thông qua các hook và đồng bộ hóa với đồ thị tri thức vĩnh viễn khi kết thúc phiên.

**Thiết lập:**

```bash
# Cài đặt cognee
pip install cognee

# Cấu hình
export LLM_API_KEY="your-openai-key"

# Clone plugin
git clone https://github.com/topoteretes/cognee-integrations.git

# Kích hoạt plugin (thêm vào ~/.zshrc để sử dụng lâu dài)
claude --plugin-dir ./cognee-integrations/integrations/claude-code
```

Hoặc kết nối với Cognee Cloud thay vì chạy cục bộ:

```bash
export COGNEE_SERVICE_URL="https://your-instance.cognee.ai"
export COGNEE_API_KEY="ck_..."
```

Plugin sẽ móc nối vào vòng đời của Claude Code — `SessionStart` khởi tạo bộ nhớ, `PostToolUse` ghi lại các hành động, `UserPromptSubmit` tiêm vào ngữ cảnh liên quan, `PreCompact` bảo toàn bộ nhớ qua các lần đặt lại ngữ cảnh, và `SessionEnd` kết nối dữ liệu phiên vào đồ thị vĩnh viễn.

### Kết nối tới Cognee Cloud

Trỏ bất kỳ agent Python nào vào một phiên bản Cognee được quản lý — tất cả các lệnh gọi SDK sẽ định tuyến đến cloud:

```python
import cognee

await cognee.serve(url="https://your-instance.cognee.ai", api_key="ck_...")

await cognee.remember("ngữ cảnh quan trọng")
results = await cognee.recall("chuyện gì đã xảy ra?")

await cognee.disconnect()
```

**Trường hợp sử dụng 1 — Agent Hỗ trợ Khách hàng**

```python
Mục tiêu: Giải quyết vấn đề của khách hàng sử dụng dữ liệu cá nhân của họ xuyên suốt lịch sử tài chính, hỗ trợ và sản phẩm.

Người dùng: "Hóa đơn của tôi có vẻ sai và vấn đề vẫn chưa được giải quyết."

Cognee theo dõi: các tương tác trong quá khứ, các hành động thất bại, các trường hợp đã giải quyết, lịch sử sản phẩm

# Phản hồi của Agent:
Agent: "Tôi tìm thấy 2 trường hợp lập hóa đơn tương tự đã được giải quyết vào tháng trước.
        Vấn đề do độ trễ đồng bộ giữa hệ thống thanh toán
        và hệ thống hóa đơn — một bản vá đã được áp dụng vào tài khoản của bạn."

# Những gì diễn ra ở hậu trường:
- Hợp nhất dữ liệu từ các kênh khác nhau của công ty
- Tái tạo dòng thời gian tương tác và theo dõi kết quả
- Truy xuất các trường hợp đã giải quyết tương tự
- Lập bản đồ cho chiến lược giải quyết tốt nhất
- Cập nhật bộ nhớ sau khi thực thi để agent không bao giờ lặp lại cùng một sai lầm
```

**Trường hợp sử dụng 2 — Chắt lọc Kiến thức Chuyên gia (Copilot SQL)**

```python
Mục tiêu: Giúp các nhà phân tích cấp dưới giải quyết các tác vụ bằng cách tái sử dụng các truy vấn cấp chuyên gia, mô hình và suy luận.

Người dùng: "Làm thế nào để tính tỷ lệ giữ chân khách hàng cho tập dữ liệu này?"

# Phản hồi của Agent:
Agent: "Dưới đây là cách các nhà phân tích cấp cao đã giải quyết một truy vấn giữ chân tương tự.

- Lập bản đồ lược đồ hiện tại với các cấu trúc đã thấy trước đây
- Truy xuất các tác vụ tương tự và các bản triển khai thành công của chúng
- Điều chỉnh suy luận của chuyên gia vào ngữ cảnh hiện tại
- Cập nhật bộ nhớ với các mẫu thành công mới để các nhà phân tích cấp dưới thực hiện ở mức gần như chuyên gia
```

## Triển khai Cognee

Sử dụng [Cognee Cloud](https://www.cognee.ai) cho trải nghiệm được quản lý hoàn toàn, hoặc tự lưu trữ với một trong những cấu hình triển khai 1-click dưới đây.

| Nền tảng | Tốt nhất cho | Lệnh |
|----------|----------|---------|
| **Cognee Cloud** | Dịch vụ được quản lý, không cần duy trì cơ sở hạ tầng | [Đăng ký](https://www.cognee.ai) hoặc `await cognee.serve()` |
| **Modal** | Không máy chủ, tự động thay đổi quy mô, khối lượng công việc GPU | `bash distributed/deploy/modal-deploy.sh` |
| **Railway** | PaaS đơn giản nhất, Postgres gốc | `railway init && railway up` |
| **Fly.io** | Triển khai biên (Edge deployment), volume lưu trữ liên tục | `bash distributed/deploy/fly-deploy.sh` |
| **Render** | PaaS đơn giản với Postgres được quản lý | Nút Triển khai lên Render |
| **Daytona** | Sandbox trên Cloud (SDK hoặc CLI) | Xem `distributed/deploy/daytona_sandbox.py` |

Xem thư mục [`distributed/`](distributed/) về các script triển khai, cấu hình worker và các chi tiết bổ sung.

## Nghiên cứu & Trích dẫn

Gần đây chúng tôi đã xuất bản một bài nghiên cứu về việc tối ưu hóa đồ thị tri thức cho suy luận của LLM:

```bibtex
@misc{markovic2025optimizinginterfaceknowledgegraphs,
      title={Optimizing the Interface Between Knowledge Graphs and LLMs for Complex Reasoning},
      author={Vasilije Markovic and Lazar Obradovic and Laszlo Hajdu and Jovan Pavlovic},
      year={2025},
```
